我们非常兴奋地宣布 **DocTranslate 1.5.0** 正式发布！本次更新是我们迄今为止最具雄心的一次飞跃，旨在彻底改变开发者处理多语言文档的体验。核心在于引入了**上下文感知的AI翻译引擎**，它不仅能够智能地翻译您的YAML和Markdown文件，还能理解技术文档的独特语境，确保翻译结果既准确又自然。

## 🚀 全新特性

本次版本带来了四项激动人心的核心新功能，将您的工作效率提升到新的高度。

### AI-Powered Translation
告别生硬的逐字翻译。我们集成了先进的AI模型，能够理解代码注释、技术术语乃至整个段落的上下文，为您的YAML和Markdown文件提供更准确、更流畅的翻译。

### Multi-Language Support
全球化支持从未如此简单。DocTranslate 现已原生支持超过 **20种语言**，包括中文、日文、韩文、法文、德文、西班牙文等，轻松将您的项目文档推向世界。

### Smart Content Protection
担心翻译会破坏原有的格式或代码？我们的智能内容保护机制会在翻译过程中自动识别并保留：
- 代码块 (```code```)
- 超链接
- 文件Frontmatter元数据
- 特定关键字与占位符

### Batch Processing
时间就是金钱。现在，您可以通过一条简单的命令，一次性翻译整个目录或筛选出的多个文件，极大简化了多文件项目管理流程。
```bash
doctranslate batch ./docs --target-lang ja
```

## ⚙️ 优化与改进

除了耀眼的新功能，我们在用户体验和性能上也做了大量打磨。

- **焕然一新的CLI界面**：引入了彩色输出和清晰的进度指示器，让操作过程一目了然。
- **更友好的错误处理**：错误信息现在更加详细和人性化，能快速定位问题根源。
- **多AI提供商支持**：除了默认引擎，您现在可以灵活配置并使用 **OpenAI、Kimi、DeepSeek** 等自定义AI服务提供商。
- **并行处理优化**：通过并行翻译任务，大幅提升了批量处理大量文件时的整体速度。

## 🐛 主要问题修复

我们修复了多个影响稳定性和使用体验的关键问题：
1.  **连接稳定性**：修复了与特定API提供商交互时可能出现的SSL连接问题。
2.  **编码处理**：彻底解决了处理非ASCII字符（如特殊符号、emoji）时出现的编码错乱问题。
3.  **内存管理**：修复了在批量翻译大型文件过程中可能出现的内存泄漏，提升了工具的长时运行稳定性。
4.  **格式兼容性**：纠正了之前版本中处理复杂嵌套YAML结构时可能出现的解析错误。

## ⚠️ 重要变更说明

为了未来更清晰的结构和更强的扩展性，本次版本包含以下**破坏性变更**，升级时请特别注意：

- **CLI命令重构**：命令行接口的结构已经重新设计。例如，原来的 `doctranslate --file` 模式可能已变更。**请务必查阅最新文档并更新您的自动化脚本。**
- **配置格式迁移**：配置文件格式已从 **JSON 统一改为 YAML**，提供了更好的可读性和注释支持。升级后，旧配置文件将不再兼容。

## 📦 升级指南

升级到最新版本非常简单：
```bash
# 如果您使用pip安装
pip install --upgrade doctranslate

# 升级后，请务必备份并转换您的旧配置文件
doctranslate config migrate
```
**强烈建议**所有用户升级至1.5.0版本，以享受全新的AI翻译能力、更广泛的语言支持以及显著的性能提升。升级后，请参考我们更新后的[官方文档](https://docs.doctranslate.example.com)了解新的命令用法和配置格式。
