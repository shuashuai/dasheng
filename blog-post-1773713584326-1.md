我们非常兴奋地宣布 **DocTranslate 1.5.0** 正式发布！本次更新是我们迈向智能化、多语言文档处理平台的重要里程碑。核心在于利用先进的 AI 技术，为开发者、技术文档工程师和内容创作者提供**精准、高效且安全**的文档翻译解决方案，彻底革新您的文档本地化工作流。

## 🚀 全新特性

本次版本带来了四项激动人心的核心新功能，旨在提升您的翻译体验和效率。

### AI-Powered Translation
告别生硬的逐字翻译。我们集成了上下文感知的 AI 翻译引擎，能够智能理解 YAML 和 Markdown 文件中的技术术语、语境和逻辑，生成更准确、更符合技术文档风格的译文。

### Multi-Language Support
全球化支持从未如此简单。DocTranslate 现已原生支持 **20 多种语言**，包括但不限于中文、日语、韩语、法语、德语、西班牙语等，轻松将您的项目文档覆盖到全球主要市场。

### Smart Content Protection
翻译时，您最宝贵的代码和结构必须保持原样。我们的智能内容保护机制会在翻译过程中自动识别并**完整保留代码块、超链接和 Frontmatter 元数据**，确保技术内容的绝对准确性。

### Batch Processing
效率倍增器。现在，您只需一条命令即可翻译整个目录或筛选出的多个文件，极大简化了大型项目的本地化管理流程。
```bash
doctranslate batch ./docs --lang ja,ko
```

## ⚙️ 优化与改进

除了亮眼的新功能，1.5.0 版本在用户体验、灵活性和性能上也进行了全面打磨。

*   **焕然一新的 CLI 界面**：引入了彩色输出和清晰的进度指示器，让命令行操作更加直观友好。
*   **更完善的错误处理**：提供了更详细、对用户更友好的错误信息，帮助您快速定位和解决问题。
*   **支持自定义 AI 提供商**：除了默认引擎，您现在可以灵活配置并使用 **OpenAI、Kimi、DeepSeek** 等多种 AI 服务提供商。
*   **并行处理优化**：通过优化翻译任务的并行执行机制，显著提升了批量处理大型文件时的整体速度。

## 🐛 关键问题修复

我们修复了多个影响稳定性和使用体验的问题，确保工具更加可靠：
- 修复了与特定 API 提供商连接时的 SSL 问题。
- 解决了处理非 ASCII 字符（如特殊符号）时的编码错误。
- 修复了批量翻译大文件时可能出现的内存泄漏问题。
- 修正了嵌套 YAML 结构在翻译过程中的错误处理逻辑。

## ⚠️ 破坏性变更

为了提供更清晰、更强大的配置能力，本次版本包含两项需要您注意的变更：

1.  **CLI 命令结构重构**：我们对命令行工具的命令和子命令结构进行了优化重组，使其更符合直觉。**请检查并更新您现有的自动化脚本或 CI/CD 流水线中的相关命令。**
2.  **配置文件格式迁移**：为了提升可读性和灵活性，配置文件格式已从 **JSON 迁移为 YAML**。升级后，您需要将原有的 `config.json` 转换为新的 `config.yaml` 格式，或重新生成配置文件。

## 📦 升级指南

我们强烈建议所有用户升级至 1.5.0 版本以享受全新特性和性能提升。

**通过 pip 升级：**
```bash
pip install --upgrade doctranslate
```

升级后，请务必：
1.  参考新版本文档，了解新的 CLI 命令结构。
2.  将您的旧版 JSON 配置文件手动转换为 YAML 格式，或使用 `doctranslate config init` 命令生成新配置文件后重新配置。
