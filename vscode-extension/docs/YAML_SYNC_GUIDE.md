# 🌐 YAML 智能同步指南

本文档介绍 YAML 翻译功能的智能同步机制。

## ✨ 功能特性

### 1. 智能差异分析

系统会自动对比基础文件（如 `en.yaml`）和目标文件（如 `zh-CN.yaml`），识别以下变更类型：

| 类型 | 说明 | 示例 |
|------|------|------|
| ➕ 新增 | 基础文件有，目标文件无 | 新增功能 key |
| ✏️ 修改 | 两者都有，但内容不同 | 原文文案变更 |
| 🗑️ 删除 | 基础文件无，目标文件有 | 功能下线 |
| ✅ 未变更 | 两者相同 | 无需处理 |

### 2. 三种同步模式

#### 🚀 智能同步（推荐）
- **行为**: 翻译新增 + 重新翻译修改的 + 删除多余的
- **适用**: 日常维护更新
- **优点**: 最小化 API 调用，保持同步

#### ➕ 仅增量
- **行为**: 只翻译新增的 key
- **适用**: 快速补充新内容
- **优点**: 最快，不改动已有翻译

#### ✏️ 全部重新翻译
- **行为**: 翻译所有 key
- **适用**: 质量不佳，从头开始
- **注意**: API 调用最多，耗时最长

### 3. 分批处理 + 进度显示

- 每批处理 **10 个 key**，避免 API 超时
- 实时显示进度百分比
- 支持取消操作

## 🔍 差异检测逻辑

### 简单字符串比较

```typescript
// 检测 value 是否变更
if (baseValue.trim() !== targetValue.trim()) {
  // 标记为可能需要重新翻译
}
```

### 注意事项

1. **无法区分原文变更 vs 翻译不同**
   - 系统只能检测到 value 不同
   - 无法确定是原文改了，还是翻译不同
   - 保守做法：标记为"可能修改"，由用户决定是否重译

2. **数组和对象处理**
   - 目前主要针对字符串 value
   - 嵌套对象递归处理
   - 数组按整体处理

## 📝 最佳实践

### 1. 日常更新流程

```bash
# 1. 更新基础文件 en.yaml
# 添加/修改/删除 key

# 2. 运行翻译命令
Dasheng: 语言通

# 3. 选择"智能同步"模式
# 系统自动处理差异

# 4. 检查生成的文件
# 确认翻译质量
```

### 2. 大规模变更

如果需要大规模修改原文，建议：

1. 先备份已有翻译
2. 选择"全部重新翻译"模式
3. 或使用 Git 版本控制对比

### 3. 多人协作

- 使用 Git 管理 YAML 文件
- 提交前运行翻译保持同步
- Code Review 时检查翻译变更

## ⚙️ 配置选项

### 批处理大小

默认每批 10 个 key，可在代码中调整：

```typescript
const batchSize = 10; // 根据 API 限制调整
```

### 取消操作

翻译过程中按 `Esc` 或点击取消按钮：
- 已翻译的内容会保存
- 未翻译的 key 保持原样

## 🐛 故障排除

### 问题：翻译进度卡住

**可能原因**:
- API 响应慢
- 网络问题
- key 数量过多

**解决**:
- 检查网络连接
- 减少批处理大小
- 分批多次翻译

### 问题：差异分析不准确

**可能原因**:
- 字符串包含多余空格
- 换行符不一致

**解决**:
- 使用 `.trim()` 处理
- 统一换行符为 LF

### 问题：误删已有翻译

**避免方法**:
- 使用 Git 版本控制
- 选择"仅增量"模式（不删除）
- 翻译前备份文件

## 💡 进阶用法

### 命令行参数（计划中）

```bash
# 指定同步模式
npx dasheng translate-yaml en.yaml --target ./locales --mode=smart

# 强制全量重译
npx dasheng translate-yaml en.yaml --target ./locales --mode=full

# 仅增量
npx dasheng translate-yaml en.yaml --target ./locales --mode=incremental
```

### 配置文件

```json
{
  "translation": {
    "defaultSyncMode": "smart",
    "batchSize": 10,
    "showProgress": true
  }
}
```
