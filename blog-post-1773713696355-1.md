# 如何用大圣 CLI 1.0 轻松实现文档国际化与博客自动化

你好，开发者朋友！是否曾为项目文档的多语言翻译而头疼？或者想快速将技术笔记转化为格式优美的博客？今天，我要向你介绍一个全新的效率工具——**大圣 CLI 1.0**。它刚刚发布，旨在帮你自动化处理 YAML 和 Markdown 的翻译，并一键生成博客。本教程将假设你是初次接触此工具，我会一步步带你从安装到实战，让你快速上手。

## 第一步：安装与初识大圣 CLI

首先，你需要确保本地环境已安装 Node.js（建议版本 14 或以上）。然后，打开你的终端，通过 npm 或 yarn 全局安装大圣 CLI。

```bash
npm install -g dasheng-cli
# 或
yarn global add dasheng-cli
```

安装完成后，输入以下命令验证是否成功，并查看基础帮助信息：

```bash
dasheng --version
dasheng --help
```

你应该能看到版本号（例如 `1.0.0`）和一个简洁的命令列表。这表明大圣 CLI 已经准备就绪！

## 第二步：核心功能实战演练

大圣 CLI 1.0 主要聚焦于三个核心功能：**YAML翻译**、**Markdown翻译**和**博客生成**。我们逐一探索。

### 1. 翻译 YAML 文件（例如国际化配置文件）

假设你有一个项目的多语言配置文件 `messages.yml`，内容如下：

```yaml
en:
  welcome: "Hello, World!"
  button: "Submit"
```

你想将其翻译成中文。首先，创建一个简单的配置文件 `.dashengrc.json` 来指定目标语言和翻译服务（这里以模拟的 API 为例，实际使用时需配置有效的翻译服务密钥）：

```json
{
  "translation": {
    "targetLanguage": "zh-CN",
    "service": "mock" // 实际可能是 “google”, “deepl” 等
  }
}
```

然后，在终端中运行翻译命令：

```bash
dasheng translate-yaml ./messages.yml
```

大圣 CLI 会读取你的配置，自动翻译并生成一个新文件（如 `messages.zh-CN.yml`），内容大致如下：

```yaml
zh-CN:
  welcome: "你好，世界！"
  button: "提交"
```

**实用技巧**：你可以使用 `--output` 参数指定输出路径和文件名，保持项目结构清晰。

### 2. 翻译 Markdown 文档（技术文档或文章）

对于 Markdown 文件，流程类似。假设你有一篇英文技术博客 `post-en.md`。运行：

```bash
dasheng translate-md ./post-en.md
```

工具会保留原有的 Markdown 格式（如标题、代码块、链接），仅翻译文本内容，并生成对应的中文版本 `post-en.zh-CN.md`。

**最佳实践**：在翻译 Markdown 时，建议先在源文件中用 HTML 注释 `<!-- no-translate -->` 包裹不希望被翻译的专有名词或代码术语，以保持准确性。

### 3. 一键生成博客页面

这是大圣 CLI 的亮点功能。它可以将翻译好（或原始）的 Markdown 文件，结合模板，快速生成静态博客页面。

首先，确保你有一个简单的博客模板目录（例如 `./template`），里面包含 `index.html` 和样式文件。然后，运行生成命令：

```bash
dasheng generate-blog ./posts --template ./template --output ./dist
```

这里，`./posts` 是你的 Markdown 文章集合目录，`./template` 是模板目录，`./dist` 是输出目录。大圣 CLI 会自动将文章转换为 HTML，并集成到模板中，最终在 `./dist` 里生成完整的静态网站。

**技巧提示**：你可以在模板中使用特定的占位符（如 `{{title}}`, `{{content}}`），大圣 CLI 在生成时会自动替换。详细占位符语法请参考官方文档。

## 第三步：进阶使用与整合建议

掌握了基础操作后，你可以尝试以下进阶玩法，让大圣 CLI 更好地融入你的工作流：

- **批量处理**：使用通配符一次性翻译或生成多个文件，例如 `dasheng translate-yaml ./locales/*.yml`。
- **集成到 CI/CD**：将大圣 CLI 命令写入你的 GitHub Actions 或 GitLab CI 配置文件，实现文档的自动翻译与博客部署。
- **自定义插件**：大圣 CLI 设计有扩展接口，未来你可以开发自定义插件来处理特定格式或集成其他翻译 API。
