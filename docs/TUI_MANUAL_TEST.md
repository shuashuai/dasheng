# TUI 本地手动测试指南

## 环境准备

确保已安装依赖并构建：

```bash
cd d:\Answer\jarvis\dasheng

# 安装依赖
pnpm install

# 构建项目
pnpm build
```

## 启动 TUI

### 方式一：直接运行

```bash
node dist/cli/index.js tui
```

或简写：

```bash
node dist/cli/index.js ui
```

### 方式二：使用 pnpm

```bash
pnpm start -- tui
```

### 方式三：开发模式（热重载）

```bash
pnpm dev -- tui
```

- 按 `→` 或 `Tab` 切换到上方 Tab 导航

### 2. Tab 导航测试

按 `→` 切换到不同 Tab，观察内容变化：

- **🏠 首页**: 快速入口
- **🌐 语言通**: YAML 翻译向导
- **📝 译真经**: Markdown 翻译向导
- **🎨 画皮术**: 封面生成向导
- **✍️ 妙笔生花**: 博客生成向导

### 3. 语言通 (YAML翻译) 测试

从首页按 `↓` 选中"🌐 语言通"，按 `Enter` 进入。

**步骤测试：**

1. **文件输入**
   - 提示：`📁 基础语言文件路径`
   - 输入：`.\i18n\en.yaml`
   - 按 `Enter`

2. **目标目录**
   - 提示：`📂 目标语言文件目录`
   - 直接按 `Enter` 使用默认值

3. **语言选择**
   - 使用 `↑ ↓` 选择语言
   - 按 `Enter` 确认

4. **试演模式**
   - 提示：`试演模式 (dry-run)?`
   - 按 `← →` 切换 Yes/No
   - 按 `Enter` 确认

5. **强制重译**
   - 提示：`强制重译 (force)?`
   - 选择 No，按 `Enter`

6. **确认配置**
   - 查看配置摘要
   - 选择 Yes，按 `Enter`

7. **执行**
   - 看到 Spinner 动画：`⠋ 正在试演分析...`
   - 等待结果

8. **结果展示**
   - 成功：`✅ 试演完成！`
   - 或失败：`❌ 错误信息`

9. **重新开始**
   - 选择 Yes 重新开始
   - 选择 No 停留在结果页

### 4. 画皮术 (封面生成) 测试

切换到画皮术 Tab，测试完整流程：

6. **副标题**

8. **确认**

   - 确认后开始生成

9. **查看结果**
   - 成功后会显示生成的文件路径

### 5. 键盘快捷键测试

在任意界面测试：

| 按键 | 预期行为 |
|------|---------|
| `q` | 退出 TUI |
| `Ctrl+C` | 强制退出 |
| `← →` | Tab 间切换 |
| `Tab` | 切换到下一个 Tab |
| `↑ ↓` | 列表中选择 |
| `Enter` | 确认当前选择 |
| `y` | Confirm 中选 Yes |
| `n` | Confirm 中选 No |

## 常见问题

### 1. 终端显示乱码

**解决**：使用 Windows Terminal 或 Git Bash，确保支持 UTF-8。

### 2. TUI 无法启动

**检查**：
```bash
# 检查构建是否成功
ls dist/tui/index.js

# 检查依赖
ls node_modules/ink
```

### 3. 键盘无响应

**原因**：某些终端模拟器可能不支持 ink 的键盘事件。

**解决**：尝试使用：
- Windows Terminal (推荐)
- VS Code 集成终端
- PowerShell 7+
- Git Bash

## 退出 TUI

任何时候按 `q` 或 `Ctrl+C` 即可退出。
